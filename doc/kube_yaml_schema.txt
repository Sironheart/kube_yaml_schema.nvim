1. Introduction                                          *kube-yaml-schema-intro*

`kube-yaml-schema.nvim` resolves YAML schemas for Kubernetes manifests at
session scope and updates `yamlls` via
`workspace/didChangeConfiguration`.

==============================================================================
2. Configuration                                         *kube-yaml-schema-config*

The plugin initializes automatically when loaded.

Use `vim.g.kube_yaml_schema` to configure defaults:

>
  vim.g.kube_yaml_schema = {
    kubectl_timeout_ms = 5000,
    cache_ttl_seconds = 300,
    notify = true,
  }
<

You can also call `require("kube_yaml_schema").setup(opts)`.

When `b0o/SchemaStore.nvim` is installed,
`require("kube_yaml_schema").yamlls_config()` automatically uses
`require("schemastore").yaml.schemas()` and disables the built-in `yamlls`
schema store.

==============================================================================
3. Commands                                             *kube-yaml-schema-command*

Main command:

- `:KubeYamlSchema refresh`
- `:KubeYamlSchema refresh-all`
- `:KubeYamlSchema context [auto|current|<name>]`
- `:KubeYamlSchema clear-cache`

Legacy command aliases are still available and emit deprecation notices:

- `:KubeYamlSchemaRefresh`
- `:KubeYamlSchemaRefreshAll`
- `:KubeYamlSchemaContext`
- `:KubeYamlSchemaClearCache`

==============================================================================
4. Health                                                  *kube-yaml-schema-health*

Run `:checkhealth kube_yaml_schema` to inspect:

- Configuration validity and unknown keys
- `kubectl` availability
- Cache directory path status
- `yamlls` attach status for this session
